Trong thời gian thực tập tại **AWS** với định hướng **Cloud Engineering / Data Engineering**, tôi đã tập trung thực hiện project cá nhân: **FinOps-Optimized Serverless Medallion Data Lakehouse Architecture for Customer Behavior Analytics trên AWS**.

Project giúp tôi tiếp cận quy trình xây dựng một hệ thống Data Lakehouse hoàn chỉnh, từ thu thập dữ liệu theo hai hình thức Batch Processing và Streaming Processing, xử lý dữ liệu bằng AWS Glue ETL, tổ chức dữ liệu theo kiến trúc Medallion, truy vấn bằng Amazon Athena và trực quan hóa dữ liệu thông qua Dashboard triển khai trên Amazon EC2.

Các nội dung chính tôi đã thực hiện gồm:

- Thiết kế kiến trúc Serverless Medallion Data Lakehouse trên AWS.
- Xây dựng hai pipeline xử lý dữ liệu:
  - Batch Processing.
  - Streaming Processing.
- Sử dụng Amazon S3 để xây dựng Data Lake theo mô hình Raw, Bronze, Silver và Gold.
- Phát triển các AWS Glue ETL Jobs để xử lý dữ liệu giữa các tầng.
- Xây dựng AWS Glue Data Catalog để quản lý metadata.
- Thực hiện truy vấn dữ liệu bằng Amazon Athena.
- Phát triển Dashboard phân tích dữ liệu bằng Streamlit.
- Triển khai Dashboard trên Amazon EC2.
- Viết báo cáo theo Hugo Workshop Website gồm Worklog, Proposal, Workshop, Blog, Events, Self-Evaluation và Feedback.
- Bổ sung hình ảnh minh chứng từ AWS Console cho các thành phần như Amazon S3, AWS Glue, Amazon Athena, Dashboard và Amazon EC2.

## Điểm mạnh đạt được

- Hiểu rõ quy trình xây dựng kiến trúc Data Lakehouse theo mô hình Medallion.
- Có khả năng thiết kế pipeline xử lý dữ liệu từ Data Source đến Dashboard.
- Thành thạo hơn trong việc sử dụng các dịch vụ AWS phục vụ Data Engineering.
- Biết triển khai Dashboard phân tích dữ liệu trên Amazon EC2.
- Có khả năng tổ chức tài liệu và báo cáo theo Hugo Workshop Template.
- Nâng cao kỹ năng nghiên cứu tài liệu và giải quyết các vấn đề kỹ thuật phát sinh.

## Điểm cần cải thiện

- Cần tìm hiểu sâu hơn về tối ưu hiệu năng của AWS Glue và Amazon Athena đối với dữ liệu lớn.
- Cần nghiên cứu thêm về bảo mật hệ thống, IAM Role và triển khai CI/CD cho Dashboard.
- Cần tiếp tục cải thiện khả năng tối ưu chi phí AWS (FinOps) khi vận hành hệ thống.
- Cần nâng cao kỹ năng trình bày tiếng Anh trong các tài liệu kỹ thuật.

## Kế hoạch cải thiện tiếp theo

Trong thời gian tới, tôi sẽ tập trung vào các nội dung sau:

1. Tìm hiểu sâu hơn về Data Engineering và kiến trúc Lakehouse trên AWS.
2. Nghiên cứu các giải pháp CI/CD để tự động triển khai Dashboard.
3. Tiếp tục tối ưu chi phí và hiệu năng của hệ thống trên AWS.
4. Mở rộng Dashboard với nhiều chỉ số phân tích và trực quan hóa nâng cao.
5. Tiếp tục thực hành các dự án Cloud nhằm nâng cao kinh nghiệm thực tế.

## Kết luận tự đánh giá

Qua kỳ thực tập, tôi đã có cơ hội vận dụng kiến thức đã học vào một dự án thực tế trên nền tảng AWS. Quá trình xây dựng hệ thống Customer Behavior Analytics giúp tôi hiểu rõ hơn về kiến trúc Data Lakehouse, quy trình xử lý dữ liệu, các dịch vụ AWS cũng như cách triển khai một hệ thống phân tích dữ liệu hoàn chỉnh.

Tôi tự đánh giá kết quả thực tập ở mức **Tốt**. Dự án đã giúp tôi nâng cao kiến thức chuyên môn, kỹ năng giải quyết vấn đề và kinh nghiệm triển khai hệ thống Cloud thực tế, đồng thời tạo nền tảng để tiếp tục phát triển trong lĩnh vực Cloud Computing và Data Engineering.
